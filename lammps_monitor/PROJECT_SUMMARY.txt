【核心功能】
================================================================================

✅ 1. 单步位移限制与自适应dt
  - 使用 fix dt/reset 实现
  - 自动调整时间步长，保证每步位移不超过 xmax
  - 防止数值爆炸和能量发散

✅ 2. 多层异常检测
  a) 位移异常检测 (compute displace/atom)
     - 监控原子累计位移
     - 软/硬阈值双层报警
  
  b) 单原子势能异常 (compute pe/atom)
     - 识别高能原子
     - 捕捉结构缺陷和反应热
  
  c) 单原子动能异常 (compute ke/atom)
     - 监控局部过热
     - 检测能量局部集中
  
  d) 配位数变化 (compute coord/atom - 可选)
     - 追踪局部结构突变
     - 用于反应体系的键断/键生检测

✅ 3. 在线自动停机 (fix halt)
  - 多个独立的停机条件
  - 条件满足时立即停止（error hard）
  - 保存停机时的完整上下文

✅ 4. 完整日志与数据保存
  - thermo 每步输出所有监控指标
  - dump 每 N 步保存完整原子信息
  - CSV 事件汇总表
  - JSON 元数据（用于自动处理）
  - 应急帧保存（异常时的最后安全结构）

✅ 5. Python 外层控制器
  - 自动调度 LAMMPS 子进程
  - 实时日志解析和内容检测
  - 智能决策逻辑（normal / warning / abort）
  - 事件持久化和可追踪性

【固定参数】（在 config.yaml 中配置）

dtreset:
  - every: 每 N 步调整一次
  - dtmax: 最大时间步长 (ps)
  - dtmin: 最小时间步长 (ps)
  - xmax: 每步最大位移限制 (Å)

compute 指令：
  - compute displace/atom all displace/atom
  - compute pe/atom all pe/atom
  - compute ke/atom all ke/atom
  - compute coord/atom all coord/atom cutoff X.X

reduce 指令：
  - compute reduce max c_displace_atom[4]    → c_max_displace
  - compute reduce max c_pe_atom             → c_max_pe
  - compute reduce max c_ke_atom             → c_max_ke
  - compute reduce max c_coord_atom          → c_max_coord

variables：
  - v_maxdisp = c_max_displace
  - v_maxpe = c_max_pe
  - v_maxke = c_max_ke
  - v_maxcn = c_max_coord

fix halt 条件：
  - fix halt_disp all halt 1 v_maxdisp > hard_limit error hard
  - fix halt_pe all halt 1 v_maxpe > hard_limit error hard
  - fix halt_ke all halt 1 v_maxke > hard_limit error hard
  - fix halt_coord all halt 1 v_maxcn > hard_limit error hard (可选)

thermo 输出：
  step temp press pe ke etotal vol dt v_maxdisp v_maxpe v_maxke v_maxcn

================================================================================
【决策逻辑】(utils.py 中的 make_decision())
================================================================================

输入：
  - maxdisp       (最大位移 Å)
  - vmaxpe        (最大单原子势能 kcal/mol)
  - vmaxke        (最大单原子动能 kcal/mol)
  - vmaxcn        (最大配位数)
  - 四个阈值配置 (soft / hard)

逻辑：
  if maxdisp > hard_disp or vmaxpe > hard_pe or vmaxke > hard_ke or vmaxcn > hard_cn:
      return ("abort", "specific_reason")
  elif maxdisp > soft_disp or vmaxpe > soft_pe or vmaxke > soft_ke:
      return ("warning", "reason_list")
  else:
      return ("normal", "")

返回：(status, reason_string)
  - status: "abort" | "warning" | "normal"
  - reason: 触发原因及其组合

【配置参数】(config.yaml)
================================================================================

最关键的参数（一定要调整）：

1. lammps.executable
   - 默认："lmp"
   - 改为：你的 LAMMPS 可执行文件路径
   
2. lammps.data_file
   - 默认：null
   - 改为：你的数据文件，如 "data/system.data"

3. thresholds.displacement
   - soft: 2.0   (警告)
   - hard: 5.0   (停止) ← 根据体系调整！

4. thresholds.per_atom_pe
   - soft: 50.0  (警告)
   - hard: 200.0 (停止) ← 根据体系调整！

5. thresholds.per_atom_ke
   - soft: 10.0  (警告)
   - hard: 50.0  (停止) ← 根据体系调整！

6. dtreset.xmax
   - 默认：0.1   (Å)
   - 改为：你的最大允许单步位移 ← 很重要！

【LAMMPS 输入脚本 (in.monitor.lmp)】
================================================================================

该脚本分为 12 个部分，按顺序包含：

1. 【基础设置】
   - units, atom_style, read_data, neighbor（需改）

2. 【势函数设置】⭐ 必须修改！
   - 给出 5 个常见示例（LJ, EAM, ReaxFF 等）
   - 用户需取消注释一个并修改参数

3. 【单步位移保险丝】
   - fix dt/reset with xmax, dtmax, dtmin（参数化）

4. 【温度控制】⭐ 需根据集合修改
   - NVE（微正则）/ NVT（等温）/ NPT（等压）

5. 【位移监控】
   - compute displace/atom + reduce max

6. 【势能/动能监控】
   - compute pe/atom + ke/atom + reduce max

7. 【配位数监控】（可选）
   - compute coord/atom + reduce max

8. 【键长监控】（可选，需有显式键）
   - compute property/local + reduce max

9. 【停机条件】
   - 4～5 个 fix halt（基于各异常指标）

10. 【thermo 输出】
    - every 1 step，输出所有关键量

11. 【dump 原子转储】
    - custom dumping with pe/ke/coord

12. 【主运行】
    - run N_steps 或分块 run

【特殊设计考虑】
================================================================================

✅ 多 chunk 运行支持
   - config.yaml 中配置 chunk_size 和 max_chunks
   - 可在脚本中循环调用，支持自动恢复

✅ Dry-run 模式
   - config.yaml 中 run.dry_run: true
   - 仅验证配置和输入文件，不实际运行 LAMMPS

✅ 多段日志处理
   - 自动检测多段 run 导致的重复表头
   - 只取最后一个有效段的数据

✅ 鲁棒的日志解析
   - 不依赖固定列顺序，通过列名映射
   - 能处理格式异常（如分割线、空行）

✅ 完整的错误处理
   - 日志缺失 → 报错
   - LAMMPS 崩溃 → 捕获并记录
   - 配置错误 → 验证并提示
   - 子进程超时 → 自动杀死并恢复

✅ 应急帧提取
   - 异常时自动从 dump 文件提取最后完整帧
   - 用于人工检查或作为恢复运行的初始条件

✅ 扩展友好
   - 所有决策逻辑都在 utils.py 中，易于改
   - 可加入新的 monitor 指标（修改 MonitorMetrics）
   - 可添加 webhook 报警（alert 函数预留接口）

【与其他系统的对比】
================================================================================

本系统 vs DeePMD-based 不确定性检测：
  ✓ 不需要 ML 模型，纯物理
  ✓ 无模型训练成本，立即可用
  ✓ 基于已验证的 LAMMPS 命令
  ✗ 不能检测"未见过的"化学环境

本系统 vs 人工监控脚本：
  ✓ 模块化，代码明确
  ✓ 多层决策，降低误报
  ✓ 自动保存和追踪
  ✗ 需要事先设定合理的阈值

本系统 vs 简单的 halt 条件：
  ✓ 三维度（位移/能量/配位）同时检测
  ✓ Soft/Hard 双阈值，防止误停
  ✓ Python 外层提供灵活的后处理
  ✗ 略微复杂，但仍可理解和改进

【推荐的调试流程】
================================================================================

1. 用小规模体系(~100 atoms, 低温)测试基础功能
   - 确保 LAMMPS 脚本正常运行
   - 验证 compute/thermo 输出格式正确

   - 运行几次，观察 maxdisp 等指标的分布
   - 根据历史数据调整阈值到合理范围

4. 在目标体系和条件下运行全程测试
   - 验证停机条件是否合理
   - 检查日志和元数据的生成

5. 定期备份输出的 event metadata 和 trends
   - 便于长期运行的统计分析
